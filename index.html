关于upx的脱壳全过程

Nov 17th 2024

关于upx脱壳的过程

1.加壳
首先，如果我想要脱壳，那我就需要有加了upx壳的文件，但是我没有啊，于是我就伸手向林吴扬学长要，但学长让我自己找upx，自己加壳，是为了脱壳我先学加壳，这个不难，难的是upx，对于我这个纯新来说有点难找，昊扬学长让我找承泽学长，陈哲学长让我用csdn找，找到后我也是成功把自己原先的c语言文件exe文件加壳。

2.脱壳工具的选择与学习使用
方向任务中让我们选择ida或dbg，我刚开始选择了ida，但有它点麻烦，所以我又选择了dbg，但后来发现无论ida和dbg都要使用。我跟着哔哩哔哩的教程学脱壳，一板一眼的复制他的行动，只知过程，不知理论。这个时候又被迫去csdn查了，在处一种到处乱走弯路的情况下终于基本掌握了方法与理论。

3.upx脱壳的实战
首先，打开查壳软件，将文件拖入其中，查看其属性，发现其显示upx壳。

接着，将其拖入ida中，发现他没有main主函数。

然后，又将其拖入dbg开始脱壳

f8至4个push下面

在rsp内存处设下断点


按下f9运行，停下后发现明显的4个pop和一个明显的大跳jmp

f8大跳后使用dbg插件scylla自动搜寻并转存后获得它的dump文件。


最后将他的dump拖入ida中，发现它存在main函数，脱壳成功。

4.一我的些碎碎念

这个帖子是人看的，那就应该多些人话，而不是一大堆的流水账，那谁来看啊？

脱壳这upx太简单了，当学会之后就觉得太简单了，至于我写这篇帖子所花费的时间是实操的好几倍，但这么简单的事情，走了一大堆弯，花了一大堆时间。

这里我想声明我的学习主张。我的有些同学把技术藏着掖着，别人一问问知识就十分不耐烦，那是没有一句不敷衍的话，但我却喜欢十分积极的回答别人的问题和向别人提问问题。简单的来说，他们主张单独学习，我主张合作。

我认为我们的敌人从来不是身边的同学们，因此不应该对同学们藏着掖着，相反我们应是朋友，我们应该团结一致对外，我们应当相互技术共享，相互学习优点。我教会了我的一个同学脱壳，我有好的学习资源和工具也都会与他共享，他也为我提供了很多有用的信息，我认为我们是互惠互利的。团结起来才能更加强大，而不是单打独斗，我希望能和更多的人相互合作，坦诚相见。

或许我的外观举指有些丑恶，但如果眼睛能透视的话，应该能看到我失落的心，我失落于周围的人不愿交流，我失落于心里有话没人可以倾诉，我失落于大家似乎总是以貌取人。不过我仍在寻求上进，不断的寻找朋友而不是敌人，不断的寻找合作者而不是大独狼。

希望看到帖子的人，不要乱传播，说实话，我不喜欢消息发到网上，但我似乎没什么好倾诉的地方了，同样的，我也不希望我的网址被太多人知道，或许他们会嘲笑我，这似乎与我想寻找朋友的理想相违背，但如果稍微细想，这思想它其实并不矛盾，无论如何感谢你的观看。

唉，又写多了

心路历程（后续一改）

Nov 7th 2024

搭建过程：

首先我在bilibili上找教程视频，在众多的视频中，我选择了up主神烦老狗的视频来学习使用hexo+github+zeabur搭建个人博客，然后跟着他的教程操作电脑，下载git visualcode node,使用镜向代码，穿越各种网站之间，接着遭遇各种问题，如hexo d一直提示deployer not found∶git，up 输入cnpm install -g hexo-cli显示NES不是内部命令或外部命令，最后独自通过查找，提问，搜索等方式解决所有问题。

遭遇问题：

程序无端报错，如up 输入cnpm install -g hexo-cli显示NES不是内部命令或外部命令，如hexo d一直提示deployer not found∶git；没有梯子去爬墙，家里网络很差导致各种网站老是进不去，加速器还特别难找。

解决方法：

询问up主神烦老狗（但是他不回复我），查看教程视频评论区，在网络寻找加速器（最后使用了clash作为我的加速器），使用我的移动流量。
